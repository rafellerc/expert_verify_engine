# Candidate Evaluation System — LLM-Driven Screening Agent (v1)

## 1. Objective

Design a system that evaluates whether a candidate possesses required competences while being robust to adversarial behaviors (e.g., LLM-assisted cheating).

The system estimates:

P(Acc = 1 | history)

Where:

* `Acc`: acceptance decision
* `history`: sequence of interview interactions

The system operates as a **sequential decision process with latent state estimation**.

---

## 2. System Overview

The system is a **POMDP with structured latent state**, implemented using LLMs for:

* Competence modeling
* Question generation (policy)
* Observation → evidence extraction

Core principle:

Belief_{t+1} = Belief_t + Evidence_t

---

## 3. Core Components

### 3.1 Competence Model

A competence model is generated from a role description.

Structure:

```json
{
  "competences": [
    {"name": str, "weight": float}
  ]
}
```

Properties:

* Flat structure (no hierarchy)
* Number of competences: configurable or auto-generated
* Weights normalized to sum to 1

---

### 3.2 Candidate Model (Ground Truth)

Defines the simulated candidate:

```json
{
  "competences": { "<name>": 0 | 1 },
  "behavior": "honest | cheater",
  "persona": "simple"
}
```

Used to generate a **candidate sheet** (observable profile).

---

### 3.3 Candidate Sheet

Generated by LLM and represents what the interviewer sees.

```json
{
  "summary": str,
  "experiences": [str],
  "claims": [str]
}
```

Rules:

* Honest candidates reflect true competences
* Cheaters exaggerate and may introduce inconsistencies

---

### 3.4 Belief State

Each competence is modeled as a Beta distribution:

* α_i: positive evidence
* β_i: negative evidence

Probability estimate:

p_i = α_i / (α_i + β_i)

Initial state:

α_i = 1, β_i = 1 (uninformative prior)

---

### 3.5 Evidence Model

Transforms (question, answer) into structured evidence.

#### Mode A — Raw Evidence (v1 implementation)

```json
{
  "competence": {
    "<name>": {"e_plus": float, "e_minus": float}
  },
  "behavior": {
    "cheating": float
  },
  "notes": str
}
```

#### Mode B — Factorized Evidence (future)

```json
{
  "correctness": float,
  "fluency": float,
  "confidence": float,
  "diagnosticity": float
}
```

With:

e⁺ = correctness × confidence × diagnosticity
e⁻ = (1 - correctness) × confidence × diagnosticity

Configurable via:

```
EVIDENCE_MODE = "raw" | "factorized"
```

---

### 3.6 Belief Update

For each competence:

α_i ← α_i + e_i⁺
β_i ← β_i + e_i⁻

---

### 3.7 Action Model (Question Policy)

LLM selects next question based on:

* Competence model
* Candidate sheet
* Current belief state
* Conversation history

Output:

```json
{
  "question": str,
  "target_competences": [str],
  "type": "technical | behavioral | probing"
}
```

Properties:

* Uses question templates
* Targets uncertain competences
* Can optionally include reasoning

Config flag:

```
ENABLE_REASONING = True | False
```

---

### 3.8 Decision Model

Decision is deterministic:

score = Σ w_i * p_i

accept = score > threshold

Configurable:

* threshold
* competence weights

---

### 3.9 Explanation Model (Separate)

A separate LLM generates explanation post-hoc using:

* conversation history
* belief trajectory
* final decision

This ensures:

* interpretability
* reproducibility
* separation of concerns

---

## 4. Interaction Loop

At each step:

1. Generate question (LLM)
2. Candidate answers (human or simulator)
3. Observation model extracts evidence (LLM)
4. Update belief state (deterministic)
5. Decide whether to continue or stop (LLM-driven)

Loop continues until:

* LLM outputs: `"I'm done"`

---

## 5. State Representation

### Maintained separately:

#### Conversation History

* Pure text (interviewer + candidate)
* No belief or system metadata

#### Belief State

```json
{
  "<competence>": probability
}
```

#### Full Internal State

* belief parameters (α, β)
* evidence history
* actions taken

### 6.2 Candidate Generator

### 6.3 Action Generator

* Output: next question

### 6.4 Observation Model

* Input: question + answer

All prompts:

* enforce JSON output
* no free-form text
