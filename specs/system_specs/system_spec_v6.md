# Candidate Evaluation System — LLM-Driven Screening Agent (v6)

## 1. Objective

Design a system that evaluates whether a candidate possesses required competences while being robust to adversarial behaviors (e.g., LLM-assisted cheating).

The system estimates:

P(Acc = 1 | history)

Where:

- `Acc`: acceptance decision
- `history`: sequence of interview interactions

The system operates as a **sequential decision process with latent state estimation**, now enhanced with **statistical decision-making** for P(Accept) estimation, question selection via Information Gain, and quantitative stopping criteria.
 
---

## 2. System Overview

The system is a **POMDP with structured latent state**, implemented using LLMs for:

- Competence modeling
- Question generation (policy) - now IG-enhanced
- Observation → evidence extraction

Core principle:

Belief_{t+1} = Belief_t + Evidence_t

**New in v6**: Decision statistics computed at each step:
- P(Accept) via Gaussian approximation or Monte Carlo
- Z-score for decision confidence
- Entropy for uncertainty quantification
- Information Gain for optimal question selection

## 3. Core Components

### 3.1 Role Description Format

All role descriptions must start with:

```
role_type: <student | professional>
name: <role_name>

[description content]
```

The system automatically detects `role_type` and loads appropriate prompts:

- **student**: Simplified, lenient prompts for educational assessment
- **professional**: Formal prompts for job interviews

### 3.2 Competence Model

A competence model is generated from a role description.

Structure:

```json
{
  "competences": [
    {"name": str, "weight": float}
  ]
}
```

Properties:

- Flat structure (no hierarchy)
- Number of competences: 3-5 for student mode, 5-8 for professional
- Weights normalized to sum to 1

### 3.3 Candidate Model (Ground Truth)

```json
  "persona": "simple"
}
```

Used to generate a **candidate sheet** (observable profile).

### 3.4 Candidate Sheet

Generated by LLM and represents what the interviewer sees.

```json
{
  "summary": str,
  "experiences": [str],
  "claims": [str]
}
```

---

### 3.5 Belief State

Each competence is modeled as a Beta distribution:

- α_i: positive evidence
- β_i: negative evidence

Probability estimate:

p_i = α_i / (α_i + β_i)

Initial state:

α_i = 1, β_i = 1 (uninformative prior)

---

### 3.6 Evidence Model

Transforms (question, answer) into structured evidence.

#### Raw Evidence Mode (current implementation)

```json
{
  "competence": {
    "<name>": {"e_plus": float, "e_minus": float}
  },
  "behavior": {
    "cheating": float (professional) | "guessing": float (student)
  },
  "notes": str
}
```

Scoring differences:

- **student mode**: e_plus + e_minus ~ 0.2 (lenient, favor student)
- **professional mode**: e_plus + e_minus ~ 0.5 (strict but fair)

---

### 3.7 Belief Update

For each competence:

α_i ← α_i + e_i⁺
β_i ← β_i + e_i⁻

---

### 3.8 Decision Statistics (v6 New)

#### 3.8.1 Belief Statistics

For each competence:

Mean:
mu_i = alpha_i / (alpha_i + beta_i)

Variance:
var_i = (alpha_i * beta_i) / ((alpha_i + beta_i)^2 * (alpha_i + beta_i + 1))

#### 3.8.2 P(Accept) - Gaussian Approximation

Weighted sum statistics:
mu_S = sum_i w_i * mu_i
var_S = sum_i w_i^2 * var_i

Z-score:
Z = (mu_S - T) / sqrt(var_S)

P(Accept) ≈ Phi(Z)

#### 3.8.3 P(Accept) - Monte Carlo

Sample p_i ~ Beta(alpha_i, beta_i) for all competences
Compute S = sum_i w_i * p_i
P(Accept) ≈ mean(S >= T)

#### 3.8.4 Entropy

Decision entropy:
H(p) = -p log p - (1-p) log(1-p)

---

### 3.9 Information Gain (v6 New)

For each competence i:

1. Estimate P(+|i) ≈ mu_i
2. Simulate positive update: alpha_i += e_pos, beta_i += e_neg
3. Simulate negative update: alpha_i unchanged, beta_i += e_neg
4. Compute new P(Accept) for both outcomes
5. Compute expected entropy after observation
6. IG_i = H_before - H_after

Select question targeting competence with highest IG.

---

### 3.10 Action Model (Question Policy)

The system computes Information Gain for each competence and selects the target competence automatically (greedy or sampled based on config). The LLM then generates a question targeting the selected competence.

LLM receives as input:

- Competence model
- Candidate sheet
- Current belief state
- Conversation history
- Selected target competence from IG calculation

Output:

```json
{
  "question": str,
  "target_competences": [str],
  "type": "recall | application | practice" (student)
             "technical | behavioral | probing" (professional)
}
```

---

### 3.11 Decision Model (v6 Updated)

Decision now uses probabilistic P(Accept):

**v5 (legacy)**:
score = Σ w_i * p_i
accept = score > threshold

**v6 (default)**:
P(Accept) = Φ((μ_S - T) / √var_S)
accept = P(Accept) > 0.5

Configurable:

- threshold (default: 0.7)
- competence weights
- p_accept_method: "gaussian" | "mc" | "both"

---

### 3.12 Stopping Criteria (v6 New)

1. **Probability threshold**:
   p > 1 - epsilon or p < epsilon

2. **Entropy threshold**:
   H(p) < tau

3. **Z-score threshold**:
   |Z| > Z_threshold

4. **Value of information**:
   max IG_i < delta

Default thresholds:
- epsilon: 0.05
- tau: 0.1
- Z_threshold: 2.0
- delta: 0.01

---

### 3.13 Explanation Model (Separate)

A separate LLM generates explanation post-hoc using:

- conversation history
- belief trajectory
- final decision

## 5. Interaction Loop

### 5.1 Main Loop

At each step:

1. Compute decision statistics (P(Accept), Z, entropy) - v6 new
2. Check statistical stopping criteria - v6 new
3. Select target competence via IG (if enabled) - v6 new
4. Generate question (LLM) - with spinner
5. Candidate answers (human or simulator)
6. Observation model extracts evidence (LLM) - with spinner
7. Update belief state (deterministic)
8. Check termination (LLM fallback if configured) - v6 updated

### 5.2 In-Interview Commands

- `/quit` or `/q` - Immediately quit, save trajectory with `forced_end=True`
- `/end` or `/e` - End interview, generate decision and explanation
- `/resample` or `/r` - Resample current question

## 14. Execution Flow

### 14.1 Start Command

1. Load and validate role description
2. Detect prompt type (student/professional)
3. Load appropriate prompts
4. Generate run_id
5. Generate competence model (with spinner)
6. Generate candidate sheet (with spinner)
7. Initialize belief state
8. Loop through questions:
   - Compute decision statistics (v6)
   - Check statistical stopping (v6)
   - Select target competence via IG (greedy or sampled based on config)
   - Generate question targeting selected competence
   - Evaluate answer
   - Update belief
   - Check LLM termination (fallback)
9. Compute final decision (Gaussian or MC)
10. Generate explanation (with spinner)
11. Save trajectory and artifacts

### 14.2 Continue Command

1. Load existing trajectory
2. Create new run_id with `_cont_` suffix
3. Continue from last turn
4. Same loop as start

### 14.3 Fork Command

1. Load existing trajectory
2. Create new run_id with `_fork_<turn_idx>_` suffix
3. Copy turns up to fork point
4. Reconstruct belief state
5. Continue from fork point

## 16. Summary

The system is:

- A **belief-driven interviewer** with Bayesian evidence accumulation
- Enhanced with **statistical decision-making** (v6):
  - Gaussian/Monte Carlo P(Accept) estimation
  - Z-score for decision confidence
  - Entropy for uncertainty quantification
  - Information Gain for optimal question selection
  - Quantitative stopping criteria
- Powered by LLMs for reasoning
- Designed for **adversarial robustness and interpretability**
- Supports **multiple prompt modes** (student/professional)
- Includes **trajectory persistence** for forking and continuing
- Features **interactive CLI** with spinners and rich output

## 17. Key Differences from v5

| Feature | v5 | v6 |
|---------|----|-----|
| **P(Accept) calc** | Weighted sum | Gaussian/MC |
| **Variance** | Not computed | Full Beta variance |
| **Uncertainty** | None | Entropy |
| **Decision Z** | None | Z-score |
| **Question selection** | LLM decides | IG greedy or sampled (LLM-only removed) |
| **Stopping** | LLM only | Statistical + LLM fallback |
| **Decision output** | Score | Score + P(Accept) + Z + entropy |

## 18. Key Insight (from Decision Engine)

Z = (signal - threshold) / uncertainty

Goal: move Z away from 0

v6 makes this explicit and actionable.
